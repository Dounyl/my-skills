# Architecture And Routing

## 这份文件何时读

当你需要判断：
- 哪个数据源优先
- 当前请求是否超出 skill 覆盖面
- 东财什么时候能用
- 新能力应该放在哪个目录

## 覆盖边界

- 当前最深覆盖：A 股、A 股 ETF、A 股指数、中国市场事件数据
- 当前可做的全球能力：全球股市资讯、跨市场观察、外围市场对 A 股的联动分析
- 当前未默认覆盖：没有在 references 中声明的海外交易所级深度行情、财报、逐笔或盘口接口

如果用户请求超出覆盖边界：
1. 先明确说明边界
2. 如果还能用全球资讯或市场观察回答，就继续
3. 如果用户要扩展 skill，再新增脚本和 reference，不要在回答里编造接口

## A 股数据源优先级

1. `mootdx`
2. 腾讯财经
3. 新浪 / 巨潮 / 同花顺 / 百度股市通
4. 东财

判断规则：
- 行情、K 线、实时价、盘口、部分估值：优先 `mootdx` / 腾讯
- 东财只处理独有数据：龙虎榜、解禁、两融、大宗、东财研报、东财新闻、部分资金流和板块归属

## 东财相关约束

东财接口统一视为高风险源：
- 不并发
- 请求间隔至少 1 秒，批量任务提高到 1.5 到 2 秒
- 统一复用 `Session`
- 带正常 `User-Agent` 和 `Referer`
- `403` 视为风控信号，优先降频，不要盲目重试

因此：
- 东财 HTTP 逻辑统一进 `scripts/shared/eastmoney_http.py`
- 任何领域脚本都不应直接各自实现一套 `requests.get`

## 目录责任

`SKILL.md`
- 激活条件
- 范围判断
- 文件导航

`references/domain-*.md`
- 该领域有哪些能力
- 每种能力优先用哪个源
- 关键字段与注意事项
- 建议脚本或新增入口

`scripts/shared/*`
- 跨域稳定基础设施

`scripts/<domain>/*`
- 具体端点封装

## 新能力怎么加

1. 先判断属于哪个领域
2. 如果是共享逻辑，放进 `scripts/shared/`
3. 如果是领域逻辑，放进对应 `scripts/<domain>/`
4. 在对应 `references/*.md` 增加入口说明
5. 只有激活面变化时才修改 `SKILL.md`

新增非 A 股市场能力时再补一条：
- 先写清“这次新增覆盖哪个市场、什么粒度、依赖哪个源”
- 再补脚本
- 最后补路由，不要先把主文件写成全市场都支持
