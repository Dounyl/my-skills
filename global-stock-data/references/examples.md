# Examples

用这些例子校验触发面，不要逐字复述。

## 应该触发的请求

- “帮我看 600519 最近 60 天 K 线、PE/PB、北向资金和相关新闻，给个快速结论。”
- “按算力主题筛一批 A 股，第一轮先用低风险源，第二轮再补龙虎榜和解禁数据。”
- “这两天全球股市大跌，帮我先总结外围市场，再看看 A 股半导体板块受影响最大的票。”
- “下载宁德时代最近的研报 PDF，顺手整理一致预期 EPS 和评级变化。”
- “把这个股票数据 skill 改得更容易触发，主文件别太长，细节拆到 references。”

## 不应该直接假装支持的请求

- “给我拉美股全市场逐笔成交和 Level 2 深度盘口，直接调用现成脚本。”
- “把东京交易所全部个股的财报字段按现有接口批量抓下来。”

遇到这类请求时：
- 先说明当前 skill 的深度覆盖边界
- 如果用户只是要全球市场观察或新闻摘要，继续做
- 如果用户要扩展能力，转入新增脚本和 reference 的工作流
